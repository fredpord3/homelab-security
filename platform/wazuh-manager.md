# Wazuh manager

Single-node Wazuh 4.14.5 deployment (manager + indexer + dashboard) on Ubuntu Server, dual-homed to handle ethernet + WiFi-fallback for uninterrupted log collection.

## Specs

See [`../hardware/README.md`](../hardware/README.md) for hardware. Software stack:

| Component | Version |
|---|---|
| Wazuh manager | 4.14.5 |
| Wazuh indexer | 4.14.5 (single-node) |
| Wazuh dashboard | 4.14.5 |
| OS | Ubuntu Server 24.04 LTS |

## Network

| Field | Value |
|---|---|
| Hostname | `ffwazuh` |
| Static IP (ethernet) | 192.168.20.20/24 |
| Gateway | 192.168.20.1 |
| VLAN | 20 (Lab) |
| Primary interface | `enx6c6e072d25d6` (Anker USB-C 2.5GbE adapter, MAC `6c:6e:07:2d:25:d6`) |
| WiFi fallback interface | `wlo1` (SSID `Pord-Fi`, Trusted VLAN) |
| Route metrics | ethernet 100 (primary), WiFi 600 (fallback) — set via netplan |
| Netplan config | `/etc/netplan/90-NM-c9827a10-7302-4b1b-86e8-e182d126245d.yaml` |

The dual-homed design ensures continuous log collection if the wired path drops. WiFi failover lands the box on the Trusted VLAN (different subnet) — agents reach the manager by IP, so a failover does cause brief connection drops while agents reconnect; this is acceptable at lab scale.

## Hardening

### SSH

| Setting | Value |
|---|---|
| Port | 2222 |
| Root login | Disabled |
| Password auth | Disabled (key-only) |
| Allowed users | `fwaz` (admin) |
| `PermitRootLogin` | `no` |
| `PasswordAuthentication` | `no` |

### MFA

- **Dashboard**: TOTP enabled, secret in Proton Pass
- **SSH**: No TOTP; key-only auth is the second factor

### fail2ban

- 1 active jail: `sshd`
- Same posture as Proxmox host

### File Integrity Monitoring

Configured in `/var/ossec/etc/ossec.conf`:

```xml
<directories>/etc,/usr/bin,/usr/sbin</directories>
<directories>/bin,/sbin,/boot</directories>
```

Monitors credential surfaces, all binary directories, and the boot partition.

### Localfile log sources

`ossec.conf` ingests:

- `journald` (systemd journal — captures sshd, sudo, fail2ban, kernel)
- `/var/ossec/logs/active-responses.log` (Wazuh active-response outcomes)
- `/var/log/dpkg.log` (package install/remove tracking)

## UFW

Restrictive firewall posture; agent and syslog ports open inbound on the Lab subnet, SSH on 2222 from the Lab + Trusted subnets, everything else denied. See [`./ufw-rules.txt`](./ufw-rules.txt) for the active ruleset.

## Backup

Configuration backups (`/var/ossec/etc/`) and ruleset snapshots are committed to this repo. Indexer data is not currently backed up — acceptable at lab scale since alerts can be regenerated by replaying events, but planned for production-grade roadmap items.
